## 🧠 What I Learned

### 1. Patterns Repeat Everywhere
The same patterns — pointers, accumulation, division — show up again and again. Learning to recognize them is the real skill.

**Lesson:** Don't memorize code. Memorize patterns.

### 2. Understanding the "Why" > Memorizing the "How"
Binary search isn't scary. It's just "guess the number" with math. Once I understood *why* it works, the code wrote itself.

**Lesson:** Always ask why.

### 3. Ancient Algorithms Still Power Modern Code
Euclid figured out GCD over 2000 years ago. We're still using his logic today. That's beautiful.

**Lesson:** Good ideas last.

### 4. Binary Search Is Not Scary
It's just:
- Look at the middle
- If target is smaller, go left
- If target is larger, go right
- Repeat until found or range is empty

That's it.

**Lesson:** Big names don't mean big difficulty.

### 5. It's Okay to Post Late
I wanted to post this yesterday. But practicing pseudocode and dry running took time. So I posted today.

**Lesson:** Consistency > perfection. Showing up late is still showing up.
